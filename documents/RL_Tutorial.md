# IC2 核反应堆强化学习教程

## 写在前面

如果你玩过 IC2，应该知道设计一个好的核反应堆有多难。放太多燃料棒，散热跟不上就爆炸；放太少又浪费空间，发电量上不去。手工调试几十次都不一定能找到最优方案。

所以我们想：能不能让 AI 自己学会设计反应堆？试了几个方案后，终于搞定了。这篇文档记录了整个过程，包括踩过的坑和最终的解决方案。

---

## 为什么用强化学习

最开始想过用遗传算法或者暴力搜索，但问题是搜索空间太大了。9×6 的网格，每个位置 18 种组件，理论上有 18^54 种可能（虽然大部分都不合理）。就算用超算跑一年也搜不完。

强化学习的好处是不需要遍历所有可能，AI 会自己探索，逐渐学会什么样的设计好用。就像你玩游戏，一开始瞎搞，慢慢就知道哪些操作有效了。

## 环境设计

### 状态表示

AI 需要"看到"当前的反应堆长什么样。我们用了一个 18×9×6 的三维数组：
- 18 层，每层对应一种组件（燃料棒、散热片等）
- 9×6 是反应堆的网格
- 如果某个位置放了某种组件，对应位置就是 1，否则是 0

举个例子，如果第 2 行第 3 列放了单铀棒：
```python
state[1, 1, 2] = 1.0  # 单铀棒那一层
state[0, 1, 2] = 0.0  # 空槽位那一层
# 其他层也都是 0
```

这种表示方式叫 one-hot 编码，神经网络比较好处理。

### 动作设计（重点！）

这里踩了个大坑。

**第一版设计**（失败）：
让 AI 同时选择三个东西：行号、列号、组件类型。听起来很合理对吧？但实际训练时发现，AI 经常选到已经放了组件的位置，这种"无效动作"占了 80% 以上。结果 AI 学到的策略就是"一直输出同一个动作"，因为这样至少不会被惩罚太多。

**第二版设计**（成功）：
干脆简化，AI 只需要选组件类型（18 种），位置按从左到右、从上到下的顺序自动填充。这样就不存在"无效动作"了，每个选择都是有效的。

这个改动看起来简单，但效果差别巨大。训练稳定了，评估结果也正常了。

### 奖励设计

奖励函数决定了 AI 学什么。我们的目标是：发电量高、不爆炸、温度可控。

**每一步的小奖励**：
- 放燃料棒：+0.1（鼓励发电）
- 放散热片：+0.05（鼓励散热）
- 放空槽位：0（不鼓励也不惩罚）

**最后的大奖励**：
- 主要奖励：平均功率（比如 300 EU/t 就是 300 分）
- 爆炸惩罚：-1000（严重惩罚）
- 温度惩罚：温度越高扣分越多
- 稳定奖励：完整跑完 1000 ticks 加 50 分
- 效率奖励：低温高功率再加 100 分

这些数字是调出来的，不是一开始就定好的。比如爆炸惩罚最开始是 -500，发现 AI 还是经常爆炸，就改成 -1000。

## 训练过程

### 准备工作

装依赖：
```bash
pip install stable-baselines3 gymnasium numpy torch tensorboard
```

### 开始训练

最简单的命令：
```bash
python rl_train.py --timesteps 200000 --n-envs 4
```

参数说明：
- `timesteps`：训练多少步。20 万步大概 10-20 分钟，50 万步要半小时到一小时
- `n-envs`：同时跑几个环境。4 个差不多，如果 CPU 核心多可以开到 8

训练时会看到这样的输出：
```
Episode 完成: 平均功率=320.00 EU/t, 最大堆温=4500.00, 爆炸=否
Episode 完成: 平均功率=450.00 EU/t, 最大堆温=7800.00, 爆炸=否
```

功率慢慢涨上去，爆炸率慢慢降下来，就说明在学习。

### 训练多久合适

这个没有标准答案，看情况：
- 如果曲线还在上升，继续训练
- 如果已经平了，差不多可以停了
- 一般 20-50 万步够用

我自己测试时，20 万步能到 300-400 EU/t，50 万步能到 400-500 EU/t。

## 评估模型

训练完了要测试一下效果：
```bash
python evaluate_model.py rl_models/PPO_xxx/best/best_model.zip --n-episodes 20
```

注意用 `best_model.zip`，这是评估中表现最好的版本。`final_model.zip` 是训练结束时的版本，不一定最好。

输出大概是这样：
```
[Episode 1/20] 完成:
  奖励: 450.25
  平均功率: 380.50 EU/t
  最大堆温: 6500.00
  爆炸: 否

评估统计:
  平均奖励: 425.30 ± 35.20
  平均功率: 360.80 ± 45.60 EU/t
  最大功率: 480.00 EU/t
  爆炸率: 5.00%
```

好的模型应该：
- 平均功率 300 以上
- 爆炸率 10% 以下
- 能稳定跑完 1000 ticks

如果达不到，可能需要：
- 训练更久
- 调整奖励函数
- 增加并行环境数

## 常见问题

### 训练很慢

几个办法：
1. 开更多并行环境（`--n-envs 8`）
2. 减少模拟时长（`--simulation-ticks 500`，但可能影响效果）
3. 降低评估频率（`--eval-freq 20000`）

### 模型不学习

如果训练了很久，功率还是很低，可能是：
1. 学习率不对，试试 `--learning-rate 1e-4` 或 `1e-3`
2. 奖励函数有问题，检查日志里的奖励值
3. 训练步数不够，继续训练

### 评估时表现很差

确保：
1. 用的是 `best_model.zip`
2. 评估参数和训练时一致（simulation_ticks 等）
3. 如果还是不行，可能模型确实没学好，重新训练

### 想调整奖励函数

编辑 `model/rl_env.py` 里的 `_calculate_reward` 方法。比如想更重视发电量：
```python
# 把这行
reward = avg_power × 1.0
# 改成
reward = avg_power × 2.0
```

改完要重新训练。

### 训练中断了

Stable-Baselines3 会自动保存检查点，但不支持直接恢复。可以手动加载检查点继续训练：
```python
from stable_baselines3 import PPO
model = PPO.load("rl_models/PPO_xxx/checkpoints/reactor_model_40000_steps.zip")
model.learn(total_timesteps=160000)  # 继续训练剩余步数
model.save("continued_model.zip")
```

## 技术细节

### 为什么用 PPO

试过几个算法：
- DQN：不太稳定，容易崩
- A2C：还行，但没 PPO 好
- PPO：最稳定，效果也不错

PPO 的优点是对超参数不敏感，基本不用调就能用。

### 网络结构

用的是简单的 MLP（多层感知机）：
- 输入：972 维（18×9×6 展平）
- 两个隐藏层，每层 64 个神经元
- 输出：18 维（每个组件的概率）

没用 CNN 是因为反应堆的空间结构不像图像那么重要，MLP 够用了。

### 超参数

基本用的默认值：
```python
learning_rate = 3e-4
n_steps = 2048
batch_size = 64
n_epochs = 10
gamma = 0.99
```

这些是 Stable-Baselines3 调好的，一般不用改。如果要改，建议一次只改一个，看看效果。

## 后续改进方向

现在的版本能用，但还有改进空间：

1. **可视化 AI 设计**：现在只能看动作序列，如果能直接画出反应堆布局就更好了
2. **导出配置文件**：把 AI 设计导出成 YAML，可以直接用在游戏里
3. **多目标优化**：不只优化功率，还要考虑成本、安全性等
4. **课程学习**：先让 AI 学简单的设计，再学复杂的

有兴趣的话可以自己试试。

---

## 总结

整个项目最大的收获是：**动作空间的设计比算法本身更重要**。

第一版用了复杂的动作空间，结果怎么调都不行。第二版简化了动作空间，用默认参数就能训练出不错的模型。

所以如果你也在做强化学习项目，遇到训练不收敛的问题，先检查环境设计，特别是动作空间。算法和超参数反而是次要的。
