## 项目概述

本项目使用强化学习（Reinforcement Learning, RL）训练 AI 自动设计 IC2 核反应堆配置。AI 通过不断试错，学习如何放置组件以最大化发电量，同时避免反应堆过热爆炸。

### 为什么使用强化学习？

设计一个高效且安全的核反应堆是一个复杂的优化问题：
- **搜索空间巨大**：9×6 网格，每个位置可以放置 18 种组件，总共有 18^54 种可能的配置
- **约束复杂**：需要平衡发电量和散热，考虑组件间的相互作用
- **难以手工优化**：人类很难找到全局最优解

强化学习可以自动探索这个巨大的搜索空间，找到高效的反应堆设计。

### 项目特点

- **简化的动作空间**：只需选择组件类型，位置自动填充
- **智能奖励设计**：鼓励高发电量，惩罚过热和爆炸
- **稳定的训练算法**：使用 PPO（Proximal Policy Optimization）
- **实时监控**：TensorBoard 可视化训练过程

## 强化学习基础

### 核心概念

#### 1. 环境（Environment）
反应堆设计环境，包含：
- **状态（State）**：当前反应堆的组件布局
- **动作（Action）**：选择一个组件类型放置
- **奖励（Reward）**：根据发电量、温度等计算的分数

#### 2. 智能体（Agent）
AI 模型，学习如何选择动作以最大化累积奖励。

#### 3. 策略（Policy）
智能体的决策规则：给定当前状态，选择哪个动作。

#### 4. 训练过程
1. 智能体观察当前状态
2. 根据策略选择动作
3. 环境执行动作，返回新状态和奖励
4. 智能体根据奖励更新策略
5. 重复上述过程，直到策略收敛

### PPO 算法

本项目使用 PPO（Proximal Policy Optimization）算法：
- **稳定性好**：通过限制策略更新幅度，避免训练崩溃
- **样本效率高**：可以重复使用采集的数据
- **易于调参**：超参数对性能影响较小

## 环境设计

### 状态空间（Observation Space）

**格式**：3D 张量，形状为 `(18, 9, 6)`
- 18 个通道：每个通道对应一种组件类型
- 9×6 网格：反应堆的布局
- 值为 0 或 1：表示该位置是否放置了对应组件

**示例**：
```python
# 如果位置 (1, 2) 放置了单铀棒（索引1）
observation[1, 1, 2] = 1.0  # 单铀棒通道
observation[0, 1, 2] = 0.0  # 空槽位通道
# 其他通道也为 0
```

这种表示方式称为 **one-hot 编码**，适合神经网络处理。

### 动作空间（Action Space）

**格式**：离散动作空间，范围 `[0, 17]`
- 每个动作对应一种组件类型
- 位置按从左到右、从上到下的顺序自动填充

**组件索引对照表**：
```
0: E (空槽位)
1: U (单铀棒)
2: D (双联燃料棒)
3: Q (四联燃料棒)
4: H (散热片)
5: R (反应堆散热片)
6: C (元件散热片)
7: A (高级散热片)
8: O (超频散热片)
9: HE (热交换器)
10: RH (反应堆热交换器)
11: CH (元件热交换器)
12: AH (高级热交换器)
13: N (中子反射板)
14: TN (加厚中子反射板)
15: C10 (10k冷却单元)
16: C30 (30k冷却单元)
17: C60 (60k冷却单元)
```

**为什么这样设计？**

原始设计使用 `MultiDiscrete([9, 6, 18])`，需要同时选择行、列、组件，导致：
- 大量无效动作（选择已占用的位置）
- 学习困难（模型主要收到负反馈）

新设计只选择组件类型，位置自动填充，**完全避免了无效动作问题**。

### 奖励函数（Reward Function）

奖励函数是强化学习的核心，决定了 AI 学习的目标。

#### 中间步骤奖励（每步）
```python
if component == "E":           # 空槽位
    reward = 0.0
elif component in ["U", "D", "Q"]:  # 燃料棒
    reward = 0.1
elif component in ["H", "R", "A", "O"]:  # 散热片
    reward = 0.05
else:                          # 其他组件
    reward = 0.02
```

这些小奖励引导 AI 放置有用的组件。

#### 最终奖励（Episode 结束时）

**1. 主要奖励：发电功率**
```python
reward = avg_power × 1.0
```
平均功率越高，奖励越大。

**2. 爆炸惩罚**
```python
if exploded:
    reward = -1000.0 + avg_power × 0.1
```
爆炸会受到严重惩罚，但如果在爆炸前产生了一些电力，会给予部分奖励。

**3. 温度惩罚**
```python
heat_ratio = max_heat / 10000  # 最大堆温 / 爆炸阈值

if heat_ratio > 0.9:  # 非常危险
    reward -= 200 × (heat_ratio - 0.9)
elif heat_ratio > 0.7:  # 比较危险
    reward -= 50 × (heat_ratio - 0.7)
```
温度越接近爆炸阈值，惩罚越大。

**4. 稳定性奖励**
```python
if ticks_run >= simulation_ticks:  # 完整运行
    reward += 50.0
```
鼓励设计能稳定运行的反应堆。

**5. 效率奖励**
```python
if heat_ratio < 0.5 and avg_power > 100:
    reward += 100.0
```
鼓励低温高功率的高效设计。

### Episode 流程

1. **重置环境**：创建空反应堆
2. **循环 54 步**：
   - AI 选择一个组件类型
   - 环境将组件放置到下一个位置
   - 返回小奖励
3. **运行模拟**：模拟 1000 个 tick
4. **计算最终奖励**：根据发电量、温度等
5. **Episode 结束**

## 训练流程

### 1. 环境准备

确保已安装依赖：
```bash
pip install stable-baselines3 gymnasium numpy torch tensorboard
```

### 2. 开始训练

**基础训练**：
```bash
python rl_train.py --timesteps 200000 --n-envs 4
```

**参数说明**：
- `--timesteps 200000`：总训练步数（推荐 200k-500k）
- `--n-envs 4`：并行环境数量（4-8 个，根据 CPU 核心数）
- `--learning-rate 3e-4`：学习率（默认值通常效果不错）
- `--simulation-ticks 1000`：每次评估运行的 tick 数
- `--eval-freq 10000`：每 10000 步评估一次

**高级训练**（更多步数，更多环境）：
```bash
python rl_train.py \
    --timesteps 500000 \
    --n-envs 8 \
    --learning-rate 3e-4 \
    --simulation-ticks 1000 \
    --eval-freq 10000
```

### 3. 监控训练

**实时日志**：
```bash
tail -f rl_training.log
```

你会看到类似的输出：
```
2026-03-30 16:00:00 [INFO] Episode 完成: 平均功率=320.00 EU/t, 最大堆温=4500.00, 爆炸=否
2026-03-30 16:00:01 [INFO] Episode 完成: 平均功率=450.00 EU/t, 最大堆温=7800.00, 爆炸=否
```

**TensorBoard 可视化**：
```bash
tensorboard --logdir rl_logs
```

然后在浏览器打开 `http://localhost:6006`，可以看到：
- 训练曲线：奖励、损失函数等
- 评估结果：最佳模型的性能
- 超参数：学习率、批次大小等

### 4. 训练时长

根据硬件和参数，训练时长大致为：
- **200k 步**：约 10-20 分钟（4 个并行环境）
- **500k 步**：约 30-60 分钟（8 个并行环境）

### 5. 保存的文件

训练完成后，会生成以下文件：
```
rl_models/
└── PPO_20260330_160000/
    ├── best/
    │   └── best_model.zip      # 评估中表现最好的模型
    ├── checkpoints/
    │   ├── reactor_model_20000_steps.zip
    │   └── reactor_model_40000_steps.zip
    └── final_model.zip         # 训练结束时的模型

rl_logs/
└── PPO_20260330_160000/
    ├── evaluations.npz         # 评估历史
    └── events.out.tfevents.*   # TensorBoard 日志

rl_training.log                 # 训练日志
```

**推荐使用 `best_model.zip`**，它是评估中表现最好的模型。

## 常见问题

### Q1: 训练很慢怎么办？

**A: 优化训练速度**
1. 增加并行环境数：`--n-envs 8`（根据 CPU 核心数）
2. 减少模拟 ticks：`--simulation-ticks 500`（但可能影响性能）
3. 降低评估频率：`--eval-freq 20000`
4. 使用 GPU（如果有）：会自动检测并使用

### Q2: 模型性能不提升？

**A: 可能的原因和解决方案**

1. **训练步数不够**
   - 增加到 500k 或 1M 步
   - 观察 TensorBoard 曲线是否还在上升

2. **奖励函数设计不合理**
   - 检查日志中的 episode 奖励
   - 如果大部分是负数，可能需要调整惩罚力度

3. **学习率不合适**
   - 尝试降低学习率：`--learning-rate 1e-4`
   - 或提高学习率：`--learning-rate 1e-3`

4. **环境设置问题**
   - 检查 `simulation_ticks` 是否足够长
   - 确保反应堆参数（max_hull_heat）设置正确

### Q3: 评估时模型表现很差？

**A: 检查以下几点**

1. **使用了错误的模型**
   - 确保使用 `best_model.zip` 而不是 `final_model.zip`
   - 最终模型可能过拟合或欠拟合

2. **评估参数不一致**
   - 确保评估时的 `simulation_ticks` 与训练时一致
   - 检查反应堆尺寸（rows, cols）是否匹配

3. **模型确实没学好**
   - 重新训练，增加训练步数
   - 调整超参数

### Q4: 如何调整奖励函数？

**A: 修改 `model/rl_env.py` 中的 `_calculate_reward` 方法**

例如，如果想更重视发电量：
```python
# 原来
reward = avg_power × 1.0

# 修改为
reward = avg_power × 2.0  # 加倍发电量奖励
```

或者降低爆炸惩罚：
```python
# 原来
reward = -1000.0

# 修改为
reward = -500.0  # 减少惩罚
```

修改后需要重新训练模型。

### Q5: 训练中断了怎么办？

**A: 从检查点恢复**

训练会自动保存检查点，但 Stable-Baselines3 不支持直接恢复训练。

**解决方案**：
1. 加载检查点模型
2. 继续训练剩余步数

```python
from stable_baselines3 import PPO

# 加载检查点
model = PPO.load("rl_models/PPO_xxx/checkpoints/reactor_model_40000_steps.zip")

# 继续训练
model.learn(total_timesteps=160000)  # 剩余步数

# 保存
model.save("rl_models/PPO_xxx/continued_model.zip")
```

### Q6: 如何在游戏中使用 AI 设计？

**A: 手动重建布局**

1. 运行评估并记录动作序列
2. 按照动作顺序，从左到右、从上到下放置组件
3. 在游戏中测试

## 技术细节

### 网络架构

本项目使用 **MLP（多层感知机）** 策略：
- 输入：18×9×6 = 972 维向量（展平后的状态）
- 隐藏层：2 层，每层 64 个神经元
- 激活函数：ReLU
- 输出：18 维向量（每个组件的概率）

### PPO 超参数

```python
PPO(
    "MlpPolicy",
    env,
    learning_rate=3e-4,      # 学习率
    n_steps=2048,            # 每次更新前采集的步数
    batch_size=64,           # 批次大小
    n_epochs=10,             # 每次更新的训练轮数
    gamma=0.99,              # 折扣因子
    gae_lambda=0.95,         # GAE 参数
    clip_range=0.2,          # PPO 裁剪范围
)
```

这些是经过调优的默认值，通常不需要修改。

### 训练技巧

1. **并行环境**：使用多个环境并行采集数据，提高样本效率
2. **评估回调**：定期评估模型，保存最佳版本
3. **检查点**：定期保存模型，防止训练中断
4. **日志记录**：详细记录训练过程，便于调试
